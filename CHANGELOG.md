# 更新日志 - 灵动岛 (Dynamic Island)

## nextgen-alpha1 (2026-05-02)

### 🚀 架构大版本迁移
- **渲染引擎更换为 Skija (Skia for Java)**：
    - **性能飞跃**：放弃基于 CPU 软栅格化程度较高的 NanoVG，转向由 Google 维护、JetBrains 优化的 Skia 硬件加速渲染引擎。
    - **渲染质量提升**：利用 Skia 先进的抗锯齿与文本渲染算法，UI 线条更圆润，字体清晰度大幅提升。
    - **零干扰 GL 状态管理**：重写了 OpenGL 状态机保护逻辑，确保 Skija 与 Minecraft (及 Sodium/Iris) 渲染环境完美共存，解决了之前版本中偶尔出现的渲染重影问题。

### 🎨 UI/UX 保持与稳定性
- **视觉一致性**：尽管底层架构彻底重写，但灵动岛、歌词 HUD、设置界面等所有视觉元素均保持了 v3.5 的高水准设计。
- **开箱即用与崩溃修复**：
    - **自动打包 Native 库**：Skija 的所有 Native 运行时库（Windows x64）现已自动打包。
    - **修复 Kotlin 运行时缺失**：修复了因缺少 Kotlin 标准库导致的 `NoClassDefFoundError` 启动崩溃，现在已将必要的 Kotlin 运行时完整打包进 Jar。

## v3.5 (2026-05-01)

### ✨ 新特性
- **UI 视觉精简**：
    - **状态标签下放**：将“RESPAWNING...”状态完全迁移至外部通知窗 (Badge)，主岛仅保留核心的“请停止移动”警告，进一步降低主岛视觉负担。

### 🔄 逻辑回滚
- **服务器 IP 获取策略回溯**：将服务器地址解析逻辑回滚至 v3.2 经典版本，恢复极简的 IP 提取方式与 12 字符截断规则，提升在复杂代理环境下的显示兼容性。

## v3.4 (2026-05-01)

### ✨ 新特性
- **KillSayAPI 状态显示优化**：
    - **位置迁移**：将死亡状态 (Died) 从灵动岛主体移至事件通知窗 (Badge)，减少主岛视觉负担，保持界面清爽。
    - **权重优先级系统**：Badge 现在遵循 `胜利 > 死亡 > 击杀` 的优先级逻辑，确保在多状态共存时优先显示最重要的信息。
- **资源加载机制升级**：
    - **持久化缓存**：将字体 (.ttf) 和原生库 (smtc.dll) 的解压路径从系统临时文件夹迁移至 Minecraft 目录下的 `.minecraft/dynamic-island/`，防止产生 Temp 垃圾文件。
    - **重复加载抑制**：优化了资源提取逻辑，仅在文件不存在时进行解压，提升启动速度并保护磁盘寿命。

### 🐛 Bug 修复
- **游戏开始“伪胜利”拦截**：修复了游戏开始时因状态残留导致误弹出 Victory 提示的问题。引入倒计时锁定机制，在游戏倒计时期间及结束后的 10 秒内强制屏蔽胜利通知。

## v3.2 (2026-04-30)

### ✨ 新特性
- **歌词延迟自动抵消优化**：引入指数移动平均（EMA）算法平滑网络抖动，并增加异常值过滤（Outlier Filtering）。现在歌词偏移调整更加丝滑，不再因单次网络波动导致歌词位置跳变。

### 🐛 Bug 修复
- **Minecraft 1.21.4 兼容性修复**：移除了 1.21.4 中已弃用的 `applyModelViewMatrix` 调用，解决了编译失败问题。
- **渲染状态同步优化**：重构了渲染后的 OpenGL 状态恢复逻辑，确保与 Minecraft 1.21.4 的渲染系统完美同步，消除潜在的视觉闪烁。
- **歌词进度回退支持**：修复了在歌曲中手动进度回退（Seek Back）时歌词不更新的问题。现在系统能智能识别大幅度的时间跳变并同步歌词，同时保留了针对微小抖动的锁定保护。
- **配置重置完善**：修复了设置界面“重置”按钮无法恢复歌词宽度配置的问题。

### 🎨 UI/UX 改进
- **配置项命名规范化**：将原有的模糊命名（MusicWidth）统一更名为歌词专属命名（LyricWidth），减少配置歧义。
